# Отчет по учебному проекту

## Цель работы

Разработать инструмент для рендеринга Handlebars-шаблонов из JSON-контекста и зафиксировать процесс разработки с помощью Codex.

## Текущее состояние проекта

Проект реализован в двух форматах:

1. CLI-инструмент для пакетного рендеринга шаблонов.
2. Веб-песочница для интерактивной проверки шаблонов через браузер.

## Реализованный функционал

### 1) CLI

- Поддержка аргументов `--template`, `--context`, `--out`.
- Рендер с `strict: true` в Handlebars.
- Вывод результата в stdout или запись в файл.
- Обработка типовых ошибок:
  - отсутствующие аргументы;
  - отсутствующий файл шаблона/контекста;
  - невалидный JSON;
  - ошибка компиляции или рендера шаблона.

### 2) HTTP API и веб-интерфейс

- HTTP-сервер на Node.js (`src/server.js`).
- Эндпоинт `POST /api/render` для рендера шаблона.
- Веб-страница песочницы (`public/index.html`) и React-компонент (`public/app.jsx`) с полями:
  - ввод шаблона Handlebars;
  - ввод JSON-контекста;
  - отправка на бэкенд;
  - отображение результата или ошибки.

## Архитектура

- `src/renderer.js` содержит общую логику рендера и используется и CLI, и сервером.
- `bin/hbs-runner.js` отвечает за CLI-запуск и обработку ошибок уровня командной строки.
- `src/server.js` обслуживает статические файлы и API рендера.
- `public/app.jsx` реализует клиентскую логику песочницы на React.

## Технологический стек

- Node.js
- Handlebars
- React (через UMD + JSX/Babel в браузере для учебной песочницы)
- Vitest
- CommonJS (основной код проекта)

## Проверка корректности

Критерии корректности:

- корректный рендер простых и вложенных полей;
- поддержка `each`-блоков;
- корректная обработка ошибок;
- прохождение автоматических тестов.

Результат: `npm test` проходит успешно (5 тестов из 5).

## Ограничения и возможные улучшения

Текущая версия фронтенда использует Babel в браузере (подходит для учебной песочницы, но не для production).

Возможные улучшения:
- переход на Vite/webpack-сборку React;
- расширение API (например, поддержка partials/helpers);
- добавление e2e-тестов для веб-песочницы;
- поддержка пользовательских настроек рендера.
