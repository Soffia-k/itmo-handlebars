# Отчет по учебному проекту

## Цель работы

Цель работы - разработать CLI-инструмент для генерации HTML/текста из Handlebars-шаблона и JSON-контекста, а также проанализировать процесс генерации такого инструмента с помощью Codex.

## Постановка задачи

Необходимо создать Node.js CLI, который:

- принимает путь к `.hbs`-шаблону через `--template`;
- принимает путь к `.json`-контексту через `--context`;
- опционально записывает результат по пути `--out`;
- выводит результат в stdout, если `--out` не передан;
- обрабатывает типовые ошибки ввода и рендеринга;
- покрыт тестами для основных сценариев.

## Выбранный стек

- Node.js - среда выполнения CLI.
- Handlebars - шаблонизатор.
- Vitest - тестовый фреймворк.
- CommonJS - формат модулей, выбранный для простоты учебного проекта.

## Критерии правильности

Работа считается корректной, если:

- простой шаблон успешно рендерится;
- поддерживаются вложенные поля JSON;
- поддерживается `each`-блок Handlebars;
- невалидный JSON приводит к понятной ошибке;
- результат может быть записан в файл;
- обязательные аргументы проверяются;
- отсутствующие файлы шаблона и JSON-контекста обрабатываются как ошибки;
- тесты `npm test` проходят успешно.

## Описание реализации

Проект разделен на две части:

- `src/renderer.js` - основная логика: парсинг аргументов, чтение файлов, JSON-парсинг, компиляция и рендеринг Handlebars, запись результата.
- `bin/hbs-runner.js` - CLI-обертка, которая вызывает основную функцию и выводит ошибки в stderr.

Шаблон компилируется с опцией `strict: true`. Это позволяет заметить обращение к отсутствующим полям как ошибку рендеринга, а не получить пустую строку без явного сигнала.

## Методика эксперимента

Эксперимент был организован как практическая разработка проекта с помощью Codex:

1. Сформулированы требования к CLI, тестам и документации.
2. Codex сгенерировал структуру Node.js-проекта.
3. Были созданы исходный код, тесты, пример шаблона и JSON-контекста.
4. Выполнена установка зависимостей через `npm install`.
5. Выполнен прогон тестов через `npm test`.

7. Результаты описаны в README, отчете, журнале эксперимента и презентации.

- поддержаны аргументы `--template`, `--context`, `--out`;
- добавлена обработка ошибок;
- подготовлены тесты Vitest;

Качество результата оценивалось по прохождению тестов и соответствию исходным требованиям. Дополнительный ручной сценарий запуска примера используется как проверка пользовательского пути.

## Ограничения исследования

Исследование основано на одной сессии генерации с Codex. Поэтому результаты описывают конкретный опыт разработки данного CLI-инструмента, а не универсальную оценку всех возможных задач и промптов.

Дополнительное ограничение - небольшой масштаб проекта. Для более полной оценки Codex можно повторить эксперимент на более сложном CLI: с потоковой обработкой файлов, пользовательскими helpers Handlebars, конфигурационным файлом и расширенным набором тестов.

## Вывод

В рамках работы создан рабочий CLI-инструмент для рендеринга Handlebars-шаблонов с JSON-контекстом. Codex оказался достаточным для генерации кода, тестов, примеров и учебной документации. Ключевым фактором качества стало наличие четких требований и обязательный запуск тестов после генерации.
