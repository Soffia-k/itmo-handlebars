# Презентация: Handlebars CLI + Web Sandbox и подход с Codex

Файл презентации: `presentation/handlebars-codex-analysis.pptx`.

## Слайд 1. Тема и результат проекта

Тезисы:

- создан учебный инструмент рендеринга Handlebars из JSON;
- реализованы CLI и веб-песочница;
- процесс разработки и доработок выполнен с помощью Codex.

Текст выступления:

На первом слайде фиксируется итог: проект перерос формат только CLI и сейчас включает два пользовательских сценария — автоматизированный (через консоль) и интерактивный (через браузер).

## Слайд 2. Постановка задачи

Тезисы:

- исходно требовался CLI с аргументами `--template`, `--context`, `--out`;
- обязательны обработка ошибок и тестируемость;
- дополнительно добавлена песочница для ручной проверки шаблонов.

Текст выступления:

Базовое задание было про CLI, но для удобства демонстрации и проверки шаблонов была добавлена веб-страница, которая отправляет шаблон и JSON на бэкенд и показывает результат рендера.

## Слайд 3. Архитектура решения

Тезисы:

- `bin/hbs-runner.js` — CLI-вход;
- `src/renderer.js` — общая логика рендера (переиспользуется CLI и сервером);
- `src/server.js` — HTTP-сервер и API `/api/render`;
- `public/index.html` + `public/app.jsx` — React-фронтенд песочницы.

Текст выступления:

Ключевая идея архитектуры — не дублировать рендеринг: общая функция в `renderer.js` используется и в CLI, и в веб-сценарии.

## Слайд 4. Пользовательские сценарии

Тезисы:

- CLI: пакетный рендер и вывод в stdout/файл;
- Web: ввод шаблона и JSON в форме, отправка на бэкенд;
- единая модель ошибок (`error` в ответе API и сообщения в CLI).

Текст выступления:

Этот слайд показывает, что инструмент покрывает два типа использования: автоматизацию и интерактивную отладку шаблонов.

## Слайд 5. Тестирование и корректность

Тезисы:

- тесты покрывают простой рендер, вложенные поля, `each`, ошибку JSON и запись в файл;
- `npm test` проходит успешно (5/5);
- рендер выполняется со `strict: true`.

Текст выступления:

Важно подчеркнуть, что проект проверяется не только вручную через браузер, но и автоматическими тестами на уровне основной логики.

## Слайд 6. Методика работы с Codex

Тезисы:

- требования формулировались итеративно;
- сначала реализован CLI-контур и тесты;
- затем добавлены сервер и UI;
- документация синхронизирована с текущим кодом.

Текст выступления:

Здесь акцент на воспроизводимом процессе: каждая доработка сопровождалась проверкой и обновлением артефактов проекта.

## Слайд 7. Ограничения и развитие

Тезисы:

- текущий React-фронтенд использует Babel в браузере (учебный вариант);
- возможна миграция на Vite для production-подхода;
- можно расширить API (helpers, partials, e2e-тесты).

Текст выступления:

Слайд показывает, что проект завершен в учебных рамках, но имеет понятную траекторию развития.

Тезисы:

Текст выступления:
