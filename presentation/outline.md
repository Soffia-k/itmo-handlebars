# Презентация: Handlebars CLI и Codex-подход

Файл презентации: `presentation/handlebars-codex-analysis.pptx`.

## Слайд 1. Генерация данных через Handlebars: Codex-подход

Тезисы:

- реализован рабочий Node.js CLI;
- добавлены Vitest-тесты и примеры;
- исследуется процесс генерации проекта с помощью Codex.

Текст выступления:

На первом слайде нужно обозначить предмет работы: это учебный проект про Handlebars CLI. Основной фокус - анализ того, как Codex помогает сгенерировать код, тесты, примеры и документацию.

## Слайд 2. Постановка задачи

Тезисы:

- CLI принимает шаблон `.hbs` и JSON-контекст;
- опциональный аргумент `--out` записывает результат в файл;
- ошибки ввода и рендеринга должны быть понятными.

Текст выступления:

Здесь объясняется техническое задание. Инструмент принимает шаблон, JSON-контекст и опциональный путь результата. Важная часть задания - не только успешный сценарий, но и понятные ошибки для типовых отказов.

## Слайд 3. Архитектура реализации

Тезисы:

- `bin/hbs-runner.js` является точкой входа CLI;
- `src/renderer.js` содержит основную логику;
- `strict: true` помогает обнаруживать отсутствующие поля.

Текст выступления:

На слайде показывается, что реализация не монолитная: CLI только запускает функцию, а основная логика вынесена в `renderer.js`. Это упростило тестирование и позволило проверять рендеринг без запуска отдельного процесса.

## Слайд 4. Критерии правильности и тесты

Тезисы:

- проверены простой шаблон, вложенные поля и `each`;
- проверены невалидный JSON и запись в файл;
- первый тестовый прогон выявил проблему формата Vitest-теста.

Текст выступления:

Здесь стоит подчеркнуть, что тесты не декоративные: они закрывают все обязательные сценарии из задания. Также важно показать реальную итерацию исправления: ошибка Vitest была обнаружена запуском тестов и исправлена изменением формата тестового файла.

## Слайд 5. Методика эксперимента

Тезисы:

- требования были сформулированы одним заданием;
- Codex сгенерировал код, тесты и документацию;
- проверка выполнялась через `npm install`, `npm test` и пример запуска.

Текст выступления:

Методика строилась как воспроизводимая Codex-сессия. Сначала задавались требования, затем генерировались артефакты проекта, после чего результат проверялся установкой зависимостей, тестами и ручным запуском примера.

## Слайд 6. Результаты Codex

Тезисы:

- создан CLI на Node.js без TypeScript;
- добавлены примеры шаблона, корректного и некорректного JSON;
- подготовлены README, отчет, журнал и презентация.

Текст выступления:

На слайде перечисляются конкретные артефакты проекта. Это важно для учебной защиты: результат можно открыть, запустить и проверить, а не только прочитать описание.

## Слайд 7. Анализ процесса генерации

Тезисы:

- подробное задание помогло сразу получить правильную структуру;
- тесты выявили реальную ошибку интеграции с Vitest;
- исправление было локальным и не усложнило основной CLI.

Текст выступления:

Этот слайд показывает, что анализ не ограничивается итоговым кодом. Важен сам процесс: Codex сгенерировал основу проекта, тестовый прогон обнаружил проблему, после чего была сделана точечная правка тестовой инфраструктуры.
